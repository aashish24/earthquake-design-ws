Deterministic Endpoint Documentation

Request Parameters

Required parameters

latitude
Latitude for location at which to compute composite results.
Example: 34.01
longitude
Longitude for location at which to compute composite results.
Example: -118.01
referenceDocument
Design code reference document for which to fetch data.
Example: ASCE41-13

Output

Web service response

request
date
Timestamp
Example: YYYY-MM-DDThh:mm:ssZ
referenceDocument
Identifying the reference document for the returned data.
Example: ASCE41-13
status
String (“success” or “error”)
Example: success
url
URL to reproduce this result.
Example: …/deterministic.json?latitude=34&longitude=-118&referenceDocument=ASCE41-13
parameters
latitude
Latitude for location at which to compute composite results.
Example: 34.01
longitude
Longitude for location at which to compute composite results.
Example: -118.01
response
data
pgad
Mapped deterministic acceleration value. (Peak Ground Acceleration)
ssd
Mapped deterministic acceleration value. (0.2 second)
s1d
Mapped deterministic acceleration value. (1.0 second)
metadata
dataset
Identifier for the underlying dataset in the database.
floor_pgad
Minimum constraint placed on the deterministic ground motion. (Peak Ground Acceleration)
floor_s1d
Minimum constraint placed on the deterministic ground motion. (1.0 second)
floor_ssd
Minimum constraint placed on the deterministic ground motion. (0.2 second)
percentile_pgad
Factor applyed to 50% ground motion to achieve target percentile ground motion. (Peak Ground Acceleration)
percentile_s1d
Factor applyed to 50% ground motion to achieve target percentile ground motion. (1.0 second)
percentile_ssd
Factor applyed to 50% ground motion to achieve target percentile ground motion. (0.2 second)

…/deterministic.json?latitude=34&longitude=-118&referenceDocument=ASCE41-13
